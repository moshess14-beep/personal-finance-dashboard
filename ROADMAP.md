# Roadmap — מעקב הון אישי

מסמך תכנון חי. מטרתו לשמור הקשר בין סשנים/שיחות פיתוח, כדי שכל המשך פיתוח (גם בסשן חדש) יבנה על ההחלטות הקודמות ולא יסתור אותן.

## עקרון-על: שתי שכבות נתונים נפרדות

האפליקציה בנויה משתי שכבות עצמאיות, בקצב עדכון שונה:

| שכבה | מייצגת | קצב עדכון | Snapshot |
|---|---|---|---|
| **מאזן** (Net Worth) | מה יש לי עכשיו — נכסים והתחייבויות | רבעוני (או כשיש שינוי משמעותי) | `QuarterlySnapshot` |
| **תזרים חודשי** (Cash Flow) | כמה נכנס/יוצא כל חודש — הכנסות מול הוצאות/חיסכון קבועים | חודשי | `MonthlyCashFlowSnapshot` |

הן **לא** אותו דבר ולא מתערבבות: אפשר שהמאזן יגדל בזמן שהתזרים החודשי שלילי (למשל עליית ערך נדל"ן), ולהפך. הגרפים ההיסטוריים של כל שכבה מבוססים על ה-Snapshots שלה בלבד.

---

## מצב נוכחי (V1 — הושלם ופרוס)

- מסך **נכסים** ומסך **התחייבויות** — CRUD מלא, ללא היסטוריה (המצב הנוכחי בלבד).
- מסך **חיסכון חודשי** (`/savings`) — רשימת רכיבי חיסכון קבועים (פנסיה, קרן השתלמות, חיסכון לכל ילד, החזר קרן הלוואה, הפקדה עצמאית, אחר), ללא היסטוריה חודשית עדיין (ראו "שאלה פתוחה" למטה — זה צפוי להיבלע לתוך שכבת התזרים החודשי).
- **Dashboard** — שווי נקי (Hero) עם מגמה יומית קלה (`netWorthHistory`, נרשם אוטומטית ברקע — לא Snapshot רשמי, רק כדי שתמיד תהיה מגמה להציג לפני שה-Snapshots הרבעוניים הרשמיים קיימים), כרטיסי KPI, גרפי דונאט, מד נכסים/התחייבויות.
- גיבוי/שחזור JSON, Dark Mode, GitHub Pages + תיקון ניתוב SPA.

## V2 — Snapshots רבעוניים למאזן (הבא בתור)

- כפתור "סגירת רבעון" → שומר `QuarterlySnapshot` (עותק קפוא של כל הנכסים/ההתחייבויות + סכומים).
- השוואה בין רבעונים: כמה גדל/ירד כל נכס/התחייבות בנפרד.
- גרף שווי נקי אמיתי לאורך זמן (מבוסס Snapshots, לא `netWorthHistory` הזמני).
- ה-Dashboard משתמש ב-Snapshot האחרון להצגת "שינוי מהרבעון הקודם" / "שינוי מתחילת השנה" במקום המגמה היומית הזמנית.

## V3 — שכבת "תזרים חודשי" (חדש, טרם מתוכנן בפירוט)

**הרעיון**: המשתמש מגדיר פעם אחת רשימת רכיבים קבועים (הכנסות + הוצאות/חיסכון), ובכל חודש רק מעדכן סכומים שהשתנו — לא מקליד הכל מחדש.

### מבנה נתונים מתוכנן

```
CashFlowComponent {
  id
  direction: 'income' | 'outflow'
  category: string        // ראו רשימות קטגוריות למטה, תלוי direction
  name: string             // "משכורת משה", "משכנתא"
  currentAmount: number    // "חי" — נערך במקום, נשאר קבוע מחודש לחודש עד שמשנים אותו
  note?: string
  updatedAt
}

// קטגוריות הכנסה: salary | business | rental | benefits | dividendsInterest | other
// קטגוריות הוצאה/חיסכון: mortgage | loan | pensionContribution | kerenHishtalmut | childSavings | additionalDeposit

MonthlyCashFlowSnapshot {
  id
  month: "YYYY-MM"
  income: { [componentId]: amount }    // עותק קפוא בזמן העדכון החודשי
  outflow: { [componentId]: amount }
  totalIncome, totalOutflow, netCashFlow   // מחושבים ונשמרים, לא מחושבים מחדש בכל טעינה
  createdAt
}
```

### זרימת העבודה המתוכננת

1. מסך הגדרה: רשימת `CashFlowComponent` (הכנסות והוצאות/חיסכון קבועים), עם `currentAmount` שאפשר לערוך בכל רגע — כמו מסכי נכסים/התחייבויות היום.
2. פעולת "**עדכון חודשי**": פותחת תצוגה שמראה את כל הרכיבים עם הסכום הנוכחי שלהם, המשתמש מתקן רק מה שהשתנה, ולוחץ "סגור עדכון לחודש X" → זה יוצר/מעדכן `MonthlyCashFlowSnapshot` לחודש הנוכחי (upsert לפי חודש, בדומה למנגנון `recordNetWorthSnapshot` הקיים).
3. Dashboard/מסך ייעודי: סך הכנסות, סך הוצאות קבועות, תזרים נטו (הכנסות פחות הוצאות), והשוואה לחודש הקודם — מבוססים על `MonthlyCashFlowSnapshot` history.
4. כל הגרפים העתידיים של תזרים (מגמת הכנסות/הוצאות, שיעור חיסכון מהכנסה) מבוססים על ה-Snapshots החודשיים בלבד.

### שאלה פתוחה שצריך להכריע בזמן המימוש

מסך `/savings` הקיים (V1) חופף באופן משמעותי לצד ה"הוצאה/חיסכון" של התזרים החודשי (פנסיה, קרן השתלמות, חיסכון לכל ילד — זהים; רק משכנתא/הלוואות והפקדות נוספות חדשות). **המלצה**: כשניגשים ל-V3, למזג — מסך `/savings` הופך לצד ההוצאות הקבועות בתוך מסך "תזרים חודשי" אחד (עם הכנסות בצד השני), במקום שני מסכים נפרדים שמזינים מידע דומה. לא להחליט סופית עכשיו — להכריע כשמתחילים לממש בפועל.

## V4 — סטטיסטיקות (היה V4, ללא שינוי מהותי)

מסך שמחשב על סמך נתונים קיימים (משתי השכבות): שיעור צמיחת הון, שיעור חיסכון מתוך הכנסה (עכשיו אפשרי בזכות V3!), יחס התחייבויות/נכסים, % מכל קטגוריה, הנכס/ההתחייבות הגדולים ביותר.

## V5 — תחזיות (היה V5)

עם V3 מומש, התחזית יכולה להתבסס על תזרים אמיתי (הכנסות בפועל פחות הוצאות בפועל) במקום מספר "חיסכון חודשי" יחיד שהוזן ידנית. עדיין: תשואה שנתית, אינפלציה, וטווח (5/10/20 שנה) שהמשתמש מגדיר.

## רעיונות עתידיים נוספים (עדיין לא מתוכננים לגרסה ספציפית)

- הפרדה בין סוגי צמיחת הון: חיסכון אקטיבי (מהתזרים) מול רווחי הון (עליית שווי נכסים במאזן) מול הכנסה משכירות — כרגע השינוי בשווי הנקי מערבב הכל; עם V2+V3 יש כבר את הנתונים הגולמיים להפריד, רק צריך מסך שמציג את זה.
- אחוז מהוצאות המחיה שממומן מהכנסות מנכסים (עצמאות פיננסית) — נגזר משכבת התזרים (V3) ברגע שיש בה קטגוריית "שכירות/דיבידנדים/ריבית" מול "הוצאות".
- חישוב אוטומטי של ירידת קרן הלוואה לפי ריבית ולוח סילוקין (כרגע רק סכום שמוזן ידנית בכל עדכון חודשי/רבעוני).
