# אפיון: אפליקציה אישית לניהול ספרים, סרטים וסדרות מומלצים

**גרסה:** 1.0 · **תאריך:** 11.07.2026 · **סטטוס:** מוכן לפיתוח

---

## 1. תקציר ומטרה

אפליקציה אישית (לא מסחרית) לשמירה, העשרה וסינון של המלצות תוכן: **ספרים, סרטים וסדרות**.

הרעיון המרכזי: המשתמש נתקל בהמלצה (בוואטסאפ, בכתבה, ברשת חברתית), מצלם מסך או מקליד שם — **וזהו**. האפליקציה עושה את כל השאר:

1. מזהה מהצילום/מהטקסט על איזו יצירה מדובר.
2. מסווגת אוטומטית: ספר / סרט / סדרה.
3. משלימה פרטים מדויקים ממקורות חיצוניים (ויקיפדיה, TMDB, Google Books).
4. שומרת הכול בענן — נגיש מכל מכשיר.
5. מאפשרת סינון חכם, כולל הפיצ'ר החשוב ביותר למשתמש: **"איפה הסרט/הסדרה זמינים לצפייה בשבילי עכשיו?"**

עקרון-על: **מינימום הקלדה מצד המשתמש, מקסימום דיוק מצד האפליקציה.** כשיש ספק — האפליקציה שואלת; כשאין תשובה — היא בוחרת את האפשרות הסבירה/המוכרת ביותר ומסמנת שהבחירה אוטומטית.

## 2. המשתמש ותרחישי שימוש

### פרופיל המשתמש
- משתמש יחיד (+ אולי בני משפחה), שימוש אישי מהנייד.
- קורא ספרים **בעברית בלבד**.
- צופה בסרטים/סדרות **בעברית או עם כתוביות בעברית**.
- רוצה אפליקציה פשוטה, ויזואלית ואינטואיטיבית — בלי טפסים ארוכים.

### תרחישים מרכזיים

| # | תרחיש | זרימה |
|---|--------|--------|
| 1 | קיבלתי המלצה בוואטסאפ | צילום מסך → שיתוף/העלאה לאפליקציה → האפליקציה מזהה, מעשירה ושומרת → אישור בהקשה אחת |
| 2 | שמעתי על ספר | הקלדת שם הספר בלבד → האפליקציה מציעה התאמות → בחירה → נשמר עם כל הפרטים |
| 3 | ערב פנוי, רוצה לראות משהו | סינון: "סרטים עד שעה וחצי, קומדיה רומנטית, זמינים בנטפליקס" → רשימה מיידית |
| 4 | מחפש ספר לחופשה | סינון: "ספר קליל לקריאה" או "עיון בכלכלה למתחילים" |
| 5 | סיימתי לצפות/לקרוא | סימון "נצפה/נקרא" + דירוג אישי אופציונלי |
| 6 | נכנס ממכשיר אחר | התחברות עם אותו חשבון → כל ההיסטוריה שם |

## 3. דרישות פונקציונליות

### 3.1 קליטת פריט חדש (Ingestion)

שלוש דרכי קלט, לפי סדר עדיפות למשתמש:

1. **צילום מסך / תמונה** (הדרך המרכזית) — העלאה מהגלריה או שיתוף ישיר מאפליקציה אחרת (Share Target).
2. **הקלדת שם בלבד** — שדה חיפוש אחד, בעברית או באנגלית.
3. **עריכה ידנית** — תמיד אפשר לתקן כל שדה אחרי הזיהוי (רשת ביטחון, לא הדרך העיקרית).

#### זרימת עיבוד צילום מסך

```
צילום מסך
   │
   ▼
[שלב א] חילוץ טקסט ומידע מהתמונה (מודל AI עם ראייה — Claude API)
   │      פלט: שם היצירה, סוג משוער (ספר/סרט/סדרה), יוצר/במאי אם מופיע, רמזים נוספים
   ▼
[שלב ב] חיפוש במקורות חיצוניים (לפי הסוג המשוער)
   │      ספר → Google Books + ויקיפדיה עברית
   │      סרט/סדרה → TMDB (תומך עברית) + ויקיפדיה כגיבוי
   ▼
[שלב ג] הכרעה
   │      התאמה אחת ברורה (ציון ביטחון גבוה) → ממשיכים אוטומטית
   │      2+ התאמות סבירות → מסך "האם התכוונת ל...?" עם כרטיסים להשוואה
   │      (כריכה/פוסטר, שנה, יוצר) — המשתמש בוחר בהקשה אחת.
   │      אם המשתמש לא מכריע → נבחרת האפשרות הפופולרית ביותר,
   │      והפריט מסומן בתגית "זוהה אוטומטית" שניתן לתקן בכל רגע.
   │      אף התאמה → הפריט נשמר עם הנתונים החלקיים + סטטוס "דורש השלמה".
   ▼
[שלב ד] העשרה — משיכת כל שדות המטא-דאטה (סעיף 4)
   ▼
[שלב ה] לסרט/סדרה: שליפת זמינות בפלטפורמות סטרימינג בישראל (סעיף 6)
   ▼
[שלב ו] מסך אישור מסכם (כרטיס מלא) → שמירה
```

**כלל דיוק מחייב:** האפליקציה לעולם לא שומרת בשקט פריט שהזיהוי שלו מסופק. או שהיא בטוחה, או שהיא שואלת, או שהיא מסמנת "זוהה אוטומטית / דורש השלמה".

### 3.2 היסטוריה וספרייה אישית

- מסך בית = הספרייה: כל הפריטים שהועלו אי-פעם, בתצוגת כרטיסים ויזואלית (כריכה/פוסטר).
- טאבים עליונים: **הכול / ספרים / סרטים / סדרות**.
- לכל פריט סטטוס: `רוצה לקרוא/לצפות` · `בתהליך` · `הושלם` · `נטשתי`.
- דירוג אישי (1–5 כוכבים) והערה חופשית — אופציונליים.
- מיון: תאריך הוספה, שם, שנה, דירוג.

### 3.3 סינון וחיפוש

שתי שכבות:

**א. סינון מובנה (פילטרים):**
- סוג (ספר/סרט/סדרה), ז'אנר, שנת יציאה, סטטוס, דירוג אישי.
- לסרטים: אורך (למשל "עד 60 דק'", "עד 90 דק'", "עד שעתיים").
- לסדרות: מספר עונות, סטטוס שידור (הסתיימה/משודרת).
- לספרים: מספר עמודים ("קצר עד 200", "בינוני", "ארוך"), עיון/פרוזה, רמת קריאה ("קליל"/"מעמיק" — תגית שנקבעת בהעשרה וניתנת לעריכה).
- **לסרטים/סדרות: זמינות בפלטפורמה** — "כל מה שזמין בנטפליקס", "זמין ב-yes" וכו'.
- שילוב פילטרים: "סרט · רומנטי · עד שעה וחצי · זמין בדיסני+".

**ב. חיפוש חופשי בשפה טבעית (שכבה נוספת מעל):**
- שורת חיפוש שמקבלת משפט חופשי: *"ספר עיון בכלכלה למתחילים"*, *"משהו קליל לערב"*.
- מימוש: המשפט נשלח למודל AI שמתרגם אותו לצירוף פילטרים מובנים ומריץ אותם — כך התוצאות תמיד מבוססות על הנתונים השמורים ולא "המצאה".

### 3.4 חשבון וסנכרון בין מכשירים

- הרשמה/התחברות: אימייל + סיסמה, ואפשרות "קישור קסם" (Magic Link) למייל — בלי לזכור סיסמה.
- כל הנתונים נשמרים בענן; כניסה מכל מכשיר מציגה את אותה ספרייה.
- עבודה offline בסיסית: הספרייה נשמרת גם מקומית (cache) לצפייה ללא רשת; הוספת פריטים דורשת רשת (בגלל ההעשרה).

### 3.5 שאלות הבהרה (Disambiguation)

- כאשר יש 2–4 מועמדים סבירים: מסך בחירה עם כרטיס לכל מועמד (תמונה, שם, שנה, יוצר, משפט תקציר).
- כפתור "אף אחד מאלה" → מעבר להקלדה ידנית של שם מדויק.
- ללא הכרעת משתמש (סגר את המסך): נבחר המועמד הפופולרי ביותר + תגית "זוהה אוטומטית".

## 4. מודל נתונים

### טבלת `items` (טבלה אחת לכל הסוגים, עם שדות ייעודיים)

| שדה | סוג | הערות |
|------|-----|--------|
| `id` | UUID | מפתח |
| `user_id` | UUID | בעלים (לסנכרון והרשאות) |
| `type` | enum | `book` / `movie` / `series` |
| `title_he` | text | שם בעברית (תצוגה ראשית) |
| `title_original` | text | שם מקורי (לחיפוש והצלבה) |
| `creator` | text | מחבר (ספר) / במאי (סרט) / יוצר (סדרה) |
| `year` | int | שנת הוצאה/יציאה |
| `genres` | text[] | ז'אנרים (רשימה מבוקרת + חופשי) |
| `summary` | text | תקציר קצר |
| `cover_url` | text | כריכה/פוסטר |
| `language_notes` | text | לספר: עברית; לסרט: "מדובב/כתוביות בעברית" |
| **ספר:** `pages` | int | מספר עמודים |
| **ספר:** `is_nonfiction` | bool | עיון/פרוזה |
| **ספר:** `reading_level` | enum | קליל / רגיל / מעמיק |
| **סרט:** `runtime_minutes` | int | אורך בדקות |
| **סדרה:** `seasons` | int | מספר עונות |
| **סדרה:** `is_ended` | bool | הסתיימה/משודרת |
| `status` | enum | רוצה / בתהליך / הושלם / נטשתי |
| `my_rating` | int | 1–5, אופציונלי |
| `my_note` | text | הערה אישית |
| `source_screenshot_url` | text | הצילום המקורי שהועלה (נשמר לתיעוד) |
| `identification` | enum | `confirmed` (אושר ע"י משתמש) / `auto` / `needs_review` |
| `external_ids` | jsonb | מזהי TMDB / Google Books / ויקיפדיה |
| `created_at`, `updated_at` | timestamp | |

## 5. מקורות מידע חיצוניים

| מקור | משמש ל... | הערות |
|-------|-----------|--------|
| **Claude API (ראייה)** | קריאת צילום המסך: חילוץ שם, סוג, רמזים | מדויק בהרבה מ-OCR קלאסי; מחזיר JSON מובנה |
| **TMDB API** | סרטים וסדרות: מטא-דאטה מלא, פוסטרים, ז'אנרים, אורך, עונות | חינמי לשימוש אישי; תומך בשפה עברית (`language=he-IL`) ובנתוני זמינות לפי מדינה (`watch/providers`, region=IL, מבוסס JustWatch) |
| **Google Books API** | ספרים: שם, מחבר, עמודים, תקציר, כריכה | חינמי; כיסוי סביר לספרים בעברית |
| **ויקיפדיה עברית (API)** | גיבוי והשלמה לשני הסוגים | במיוחד לספרי מקור עבריים שלא קיימים ב-Google Books |

**סדר עדיפויות:** מקור ייעודי (TMDB / Google Books) קודם; ויקיפדיה כגיבוי; מודל ה-AI מאחד את התוצאות ומכריע במקרה של סתירות. **אין שמירת נתון בלי מקור** — עדיף שדה ריק עם סימון "דורש השלמה" מנתון מומצא.

## 6. זמינות בפלטפורמות סטרימינג (הפיצ'ר המרכזי)

### פלטפורמות נתמכות (ישראל)
- נטפליקס (Netflix)
- דיסני+ (Disney+)
- yes / yes+
- Apple TV+
- Amazon Prime Video
- HOT / Next TV
- סלקום tv
- פרטנר TV / סטינגTV
- (הרשימה ניתנת להרחבה — enum פתוח)

### מימוש
1. בעת שמירת סרט/סדרה: קריאה ל-TMDB `watch/providers` עם region=IL → שמירה בטבלת `availability`.
2. **רענון תקופתי:** הזמינות משתנה (תכנים יורדים ועולים). רענון אוטומטי לכל הפריטים בסטטוס "רוצה לצפות" אחת לשבוע, ורענון ידני בכפתור על כל פריט ("בדוק זמינות עכשיו").
3. תצוגה בכרטיס הפריט: לוגואים של הפלטפורמות שבהן זמין + תאריך "נבדק לאחרונה".
4. סינון: "הצג את כל מה שזמין ב־X" — כולל שילוב עם שאר הפילטרים.

### מגבלה ידועה (חשוב!)
נתוני TMDB/JustWatch מצוינים לנטפליקס, דיסני+, Apple TV+ ואמזון, אך הכיסוי של פלטפורמות ישראליות (yes, HOT, סלקום, פרטנר) **חלקי**. פתרון בשלב 1: מציגים מה שיש + מאפשרים למשתמש לסמן ידנית "ראיתי שזה זמין ב-yes" (שדה זמינות ידני שנשמר לצד האוטומטי). בהמשך אפשר לבחון סריקה ישירה של אתרי הספקים.

## 7. דרישות לא-פונקציונליות

| תחום | דרישה |
|-------|--------|
| שפה וכיווניות | ממשק בעברית מלאה, RTL בכל המסכים |
| פלטפורמה | מותקנת על הנייד — ראו החלטת PWA בסעיף 8 |
| פשטות | הוספת פריט = פעולה אחת (צילום/שם) + אישור אחד לכל היותר |

| ביצועים | זיהוי + העשרה תוך ≤ 10 שניות מהעלאת צילום |
| פרטיות | הנתונים אישיים; גישה רק לחשבון המשתמש (Row Level Security) |
| עלות | שימוש אישי — יעד: 0 ₪/חודש (רמות חינמיות של כל השירותים) |

## 8. ארכיטקטורה וסטאק מומלץ (בנייה מהערב)

### החלטה: PWA (אפליקציית ווב מותקנת) ולא אפליקציית חנות
- מותקנת על מסך הבית בנייד (אנדרואיד ואייפון), נראית ומתנהגת כאפליקציה.
- תומכת שיתוף תמונה ישירות אליה (Share Target באנדרואיד; באייפון — העלאה מהגלריה).
- בלי חשבונות מפתח, בלי אישורי חנות — אפשר להתחיל לבנות ולהשתמש **היום**.
- אם בעתיד יידרש יותר (התראות פוש באייפון וכו') — מעבר ל-Capacitor עוטף את אותו קוד בדיוק.

### רכיבים

```
┌─ נייד: PWA ──────────────────────────────┐
│ React + Vite · RTL · עברית               │
│ ספריית UI: Tailwind CSS                  │
└──────────────┬───────────────────────────┘
               │
┌─ Backend: Supabase (חינמי) ──────────────┐
│ · Auth (אימייל+סיסמה / Magic Link)       │
│ · PostgreSQL (items, availability) + RLS │
│ · Storage (צילומי המסך שהועלו)           │
│ · Edge Functions — כל הלוגיקה הרגישה:    │
│    1. ניתוח צילום מסך (קריאה ל-Claude)   │
│    2. חיפוש והעשרה (TMDB/Books/ויקיפדיה) │
│    3. רענון זמינות שבועי (cron)          │
│    ← מפתחות ה-API נשמרים רק כאן, לא בצד  │
│      הלקוח                               │
└──────────────────────────────────────────┘
```

**למה כך:** צוות של אדם אחד, ערב אחד להתנעה — Supabase נותן חשבונות, DB מסונכרן, אחסון ופונקציות שרת בלי להקים שרת; ה-PWA רץ על כל מכשיר; וכל המפתחות (Claude, TMDB) מוגנים בצד השרת.

## 9. מסכים (UI)

1. **התחברות/הרשמה** — מסך אחד, אימייל+סיסמה או קישור למייל.
2. **הספרייה (בית)** — גריד כרטיסים ויזואלי; טאבים: הכול/ספרים/סרטים/סדרות; שורת חיפוש חופשי; כפתור פילטרים; כפתור `+` צף.
3. **הוספת פריט** — שתי אפשרויות גדולות: 📷 העלאת צילום מסך · ⌨️ הקלדת שם.
4. **"האם התכוונת ל...?"** — כרטיסי מועמדים להשוואה, בחירה בהקשה.
5. **כרטיס פריט** — פוסטר/כריכה, כל המטא-דאטה, לוגואי פלטפורמות זמינות + "נבדק לאחרונה", סטטוס, דירוג, הערה, כפתורי עריכה ו"בדוק זמינות עכשיו".
6. **פילטרים** — Bottom Sheet עם צ'יפים (סוג, ז'אנר, אורך/עמודים, פלטפורמה, סטטוס).

## 12. הגדרת "מוכן" (Definition of Done) ל-MVP

המערכת נחשבת מוכנה לשימוש כאשר, מהנייד:
1. אפשר להתחבר עם חשבון ולראות את אותה ספרייה משני מכשירים שונים.
2. העלאת צילום מסך של המלצת סרט מזהה נכון את הסרט (או שואלת), ושומרת אותו עם פוסטר, שנה, אורך, ז'אנר וזמינות סטרימינג.
3. הקלדת שם ספר בעברית שומרת אותו עם מחבר, עמודים, ז'אנר ותקציר.
4. סינון "סרטים זמינים בנטפליקס עד שעה וחצי" מחזיר תוצאות נכונות.
5. כל פריט ניתן לעריכה, לשינוי סטטוס ולדירוג.
